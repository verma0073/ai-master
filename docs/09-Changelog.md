# Versioning Strategy

The project follows Semantic Versioning.

Format:

MAJOR.MINOR.PATCH

Example:

1.0.0

Where:

- MAJOR → Significant architectural or project milestones
- MINOR → New features or completed sprint deliverables
- PATCH → Documentation updates, bug fixes, and small improvements

This changelog should be updated whenever:

- A sprint is completed

Minor documentation edits and typo fixes generally do not require a new version unless they significantly improve the repository.
